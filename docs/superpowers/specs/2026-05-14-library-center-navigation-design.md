# Vibrary 资料中心与导航体验设计

## 背景

Vibrary 当前已经具备 Windows 后端、Qdrant sidecar、Windows 桌面端和 Android 客户端的基本联机能力。现有易用性问题主要集中在两个方面：

- 文件进入资料库后，用户只能从导入结果或队列日志推断发生了什么，缺少统一可浏览、可管理的资料中心。
- Windows 左侧栏只是页面锚点集合，所有内容堆在同一个长页面里，不符合现代应用的导航习惯，也不利于后续扩展。

本轮目标是先把产品可用性补齐，不改变“后端科研核心 + 客户端工程体验”的架构边界。

## 产品原则

1. 一个 Windows 后端就是一个默认资料组。加入该后端的设备默认互信。
2. 所有可信设备都可以浏览资料组里的资料，包括文件名、类型、大小、来源设备、索引状态和可用操作。
3. 图片资料在资料中心和搜索结果中优先显示缩略图，而不是只显示文件名。
4. Windows 后端仍然是唯一的导入、持久化、embedding、Qdrant 索引和检索执行端。
5. Windows 和 Android 只是客户端。客户端可以上传、浏览、搜索、打开或下载资料，但不直接运行 embedding 或 Qdrant。
6. 本轮不引入账号体系、云同步、多后端组、多用户权限矩阵或复杂分享权限。

## 方案选择

推荐方案：在现有 FastAPI 后端增加资料中心 API，并让 Windows/Android 共用这组接口。

优点：

- 最小化架构风险，不需要拆分新的服务。
- 后端已经拥有 Asset、Library Copy、Device Ref、Index Job 等数据，资料中心只是把这些模型产品化。
- Windows 和 Android 可以共享同一套资料列表与缩略图策略，减少两端逻辑分叉。

不采用的方案：

- 只改 UI，不扩展后端 API。这样 Android 仍然拿不到完整资料库视图，无法解决“只能看日志”的问题。
- 在 Android 本地维护资料中心副本。这样会把同步一致性变复杂，且违背 Android 不承担核心索引职责的原则。

## 后端设计

新增资料中心能力：

- `GET /v1/library/assets`：列出资料库资产，支持 `device_id`、`query`、`kind`、`limit`、`offset`。
- 返回资料总数、分页信息、资料摘要、来源设备、索引任务状态、缩略图 URL、内容 URL 和当前请求设备的可用性。
- `GET /v1/assets/{asset_id}/thumbnail`：当资产是图片并且有 Windows 资料库正本时，返回可直接显示的图片响应。MVP 不做额外像素缩放，UI 以固定缩略图容器裁剪展示。

安全规则：

- 本机桌面端仍然可以直接访问后端。
- 局域网设备必须通过 Bearer token 访问。
- 可信设备可以浏览资料中心。带 `device_id` 查询时，后端仍校验请求设备与 token 绑定关系，避免设备伪装。

## Windows 桌面端设计

左侧导航改为真正的页面导航，主要页面为：

- 首页：服务状态、局域网地址、资料总览、索引与上传概况。
- 资料中心：资料库文件列表，图片使用缩略图网格/列表展示，显示来源、大小、类型、索引状态和打开入口。
- 导入：选择文件或文件夹，显示最近导入统计和索引入队数。
- 搜索：搜索框、结果列表、图片缩略图和打开策略。
- 传输：上传队列和索引队列。
- 设备：验证码、可信设备、移除设备。
- 设置：局域网、发现广播、自动索引、缓存和模型状态。

视觉方向：

- 保留 Electron + React + Vite，不是框架过时问题；问题是信息架构和 CSS 太像管理控制台。
- 使用更清晰的应用壳、紧凑导航、页面标题、状态条、资料网格和稳定尺寸缩略图。
- 避免营销页和装饰性视觉，产品定位是本地资料管理工具，应当清爽、可靠、可扫描。

## Android 客户端设计

新增“资料中心”入口：

- 配对成功后可以刷新资料库列表。
- 列表展示资料名、类型、大小、来源、索引状态。
- 图片资料显示后端缩略图。
- 保留现有配对、资料选择、队列、搜索、缓存能力。

本轮无法进行 ADB 真机调试，Android 验收以以下内容为准：

- Kotlin 单元测试覆盖资料中心接口 DTO 和 UI 文案。
- Gradle `testDebugUnitTest` 通过。
- Debug APK 可构建。

## 测试策略

后端：

- 覆盖资料中心列表、来源设备、索引状态、图片缩略图响应和可信设备访问语义。
- 继续覆盖 Windows 导入、上传完成、索引处理、搜索解析。

桌面端：

- 覆盖 `BackendClient` 新资料中心接口。
- 覆盖 Dashboard 数据聚合会加载资料中心。
- 覆盖中文导航文案更新。
- TypeScript 类型检查。

Android：

- 覆盖新增资料中心 DTO 字段和中文 UI 文案。
- 继续运行上传队列、配对发现、网络配置等既有测试。

## 验收标准

- Windows 启动后后端仍由桌面端无感启动。
- Windows 左侧导航切换的是不同页面，不再只是跳转到同一长页面的锚点。
- Windows 资料中心可以看到已经导入或手机上传的资料。
- 图片资料在 Windows 资料中心和搜索结果中显示缩略图。
- Android 配对后可以看到资料中心入口和资料列表模型，图片项具备缩略图 URL 展示能力。
- 版本号从 `0.1.4` 增加到 `0.1.5`。
- 后端测试、桌面测试、桌面类型检查、Android 单元测试和 APK 构建通过。
